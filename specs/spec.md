# 功能规格：数据文件夹选择

## 概述

允许用户通过网页界面选择数据文件夹，而不是在代码中硬编码数据路径。提高系统的灵活性和可配置性。

## 用户故事

### US-1: 数据文件夹选择
**作为** 数据分析师
**我想要** 在网页上选择数据文件夹
**以便于** 分析不同位置的数据而无需修改代码

#### 验收标准
- [ ] 筛选面板新增"数据文件夹"选择框
- [ ] 支持手动输入文件夹路径
- [ ] 支持从预设列表选择
- [ ] 切换文件夹后自动刷新可用日期
- [ ] 显示当前选择的文件夹路径

### US-2: 文件夹验证
**作为** 系统管理员
**我想要** 验证选择的文件夹是否有效
**以便于** 避免无效路径导致的错误

#### 验收标准
- [ ] 检查文件夹是否存在
- [ ] 检查文件夹是否包含 parquet 文件
- [ ] 显示文件夹中的数据文件数量
- [ ] 无效路径显示错误提示

## 功能需求

### FR-1: GET /api/folders
- 描述：获取可用文件夹列表
- 优先级：P0
- 依赖：无

### FR-2: POST /api/folders/validate
- 描述：验证文件夹路径
- 优先级：P0
- 依赖：FR-1

### FR-3: 前端组件
- 描述：文件夹选择器 UI
- 优先级：P0
- 依赖：FR-1, FR-2

### FR-4: 数据加载器
- 描述：支持动态数据目录
- 优先级：P0
- 依赖：无

## 非功能需求

### 安全
- 路径限制：只能访问 `/mnt/f/` 下的文件夹
- 防止路径遍历攻击

### 性能
- 文件夹扫描 < 1 秒
- 不阻塞主线程

## 约束条件

- 保持向后兼容（默认使用原数据目录）
- 遵循项目宪法 (constitution.md)
