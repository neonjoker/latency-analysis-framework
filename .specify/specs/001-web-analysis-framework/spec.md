# 功能规格：网页版交易链路延时数据分析框架

## 概述

基于现有代码和分析需求，构建一个模块化的网页分析框架，用于分析 hqmatch_storage 文件夹中的 parquet 文件中的交易链路延时数据。

## 用户故事

### US-1: 数据浏览与探索
**作为** 数据分析师
**我想要** 浏览和筛选延时数据
**以便于** 快速定位感兴趣的数据子集

#### 验收标准
- [ ] 支持按日期范围筛选
- [ ] 支持按柜台筛选
- [ ] 支持按延时阈值筛选
- [ ] 实时显示筛选结果数量

### US-2: 高延时聚集区检测
**作为** 系统运维人员
**我想要** 自动识别高延时聚集区
**以便于** 快速发现系统异常

#### 验收标准
- [ ] 自动检测高延时聚集区（延时 > 5ms）
- [ ] 显示聚集区时间分布
- [ ] 支持聚类分析识别相似链路
- [ ] 可视化展示聚集区热力图

### US-3: 链路归因分析
**作为** 性能工程师
**我想要** 分析延时来源
**以便于** 定位性能瓶颈

#### 验收标准
- [ ] 计算 TGW 处理延时
- [ ] 计算柜台处理延时
- [ ] 计算网络传输延时
- [ ] 生成延时归因饼图/柱状图

### US-4: 异常检测与告警
**作为** 监控人员
**我想要** 自动检测异常模式
**以便于** 及时响应问题

#### 验收标准
- [ ] 检测糖葫芦现象（连续高延时）
- [ ] 检测 microburst（短时间大量请求）
- [ ] 检测链路阻塞（tgwBackTime/tgwOutTime 比例异常）
- [ ] 支持告警阈值配置

### US-5: 报告生成
**作为** 团队负责人
**我想要** 生成分析报告
**以便于** 分享分析结果

#### 验收标准
- [ ] 支持 Markdown 格式报告
- [ ] 自动插入图表
- [ ] 支持自定义报告模板
- [ ] 支持导出 PDF/HTML

## 功能需求

### FR-1: 数据加载模块
- 描述：支持从 parquet 文件高效加载数据
- 优先级：P0
- 依赖：无

### FR-2: 数据处理模块
- 描述：数据清洗、转换、聚合
- 优先级：P0
- 依赖：FR-1

### FR-3: 分析算法模块
- 描述：延时计算、聚类、异常检测
- 优先级：P0
- 依赖：FR-2

### FR-4: 可视化模块
- 描述：图表生成、交互式展示
- 优先级：P1
- 依赖：FR-3

### FR-5: Web API 模块
- 描述：RESTful API 服务
- 优先级：P1
- 依赖：FR-2, FR-3

### FR-6: 前端界面
- 描述：用户交互界面
- 优先级：P1
- 依赖：FR-4, FR-5

## 非功能需求

### 性能
- 响应时间：API < 500ms，页面加载 < 3s
- 并发用户：支持 10+ 并发
- 数据处理：100 万行/秒

### 安全
- 不暴露敏感配置
- 文件访问限制在项目目录内
- 输入验证和清理

## 数据需求

### 输入数据
- 数据源：`/mnt/f/latency_project/*.parquet`
- 数据格式：Parquet
- 数据量：约 4GB/天

### 输出数据
- 输出格式：JSON API + 图表
- 存储位置：`/mnt/f/latency_project/data/results/`

## 约束条件

- 必须使用 pyenv + venv 管理 Python 环境
- Claude Code 只能在虚拟环境下执行
- 代码必须模块化，减少重复
- 遵循项目宪法 (constitution.md)
