# 功能规格：跨平台数据文件夹支持

## 概述

移除数据文件夹路径必须在 `/mnt/f/` 目录下的限制，支持跨平台（Windows/macOS/Linux）任意有效路径的数据文件夹选择。

## 问题描述

当前实现中，`src/api/routes.py` 硬编码了 `ALLOWED_ROOT = Path("/mnt/f")`，导致：
1. 无法在 Windows 上使用（如 `C:\data`）
2. 无法在 macOS 上使用（如 `/Users/data`）
3. 无法在 Linux 其他目录使用（如 `/home/user/data`）

## 用户故事

### US-1: 跨平台路径支持
**作为** 跨平台用户
**我想要** 使用任意有效路径作为数据源
**以便于** 在 Windows/macOS/Linux 上都能分析数据

#### 验收标准
- [ ] Windows 支持（如 `C:\latency_data`）
- [ ] macOS 支持（如 `/Users/data`）
- [ ] Linux 支持（如 `/home/user/data`）
- [ ] 移除 `/mnt/f/` 硬编码限制

### US-2: 路径安全验证
**作为** 系统管理员
**我想要** 验证路径的有效性和安全性
**以便于** 防止路径遍历攻击和无效访问

#### 验收标准
- [ ] 路径必须是绝对路径
- [ ] 路径必须存在且是目录
- [ ] 目录必须包含 parquet 文件
- [ ] 防止 `..` 路径遍历
- [ ] 防止访问系统敏感目录

### US-3: 配置文件支持
**作为** 高级用户
**我想要** 通过配置文件设置允许的根目录
**以便于** 自定义安全路径范围

#### 验收标准
- [ ] 支持配置文件设置 `allowed_roots`
- [ ] 默认允许所有绝对路径
- [ ] 可配置禁止访问的目录列表

## 非功能需求

### 安全
- 绝对路径验证
- 防止路径遍历攻击
- 禁止访问系统敏感目录（如 `/etc`, `C:\Windows`）

### 兼容性
- Windows 路径支持（反斜杠）
- Unix 路径支持（正斜杠）
- UNC 路径支持（Windows 网络路径）

### 性能
- 路径验证 < 100ms
- 不影响现有功能

## 约束条件

- 遵循项目宪法 (constitution.md)
